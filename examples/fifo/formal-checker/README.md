# Checker mode

This directory contains the SymbiYosys setup used to validate the synthesizable hardware checker generated by the SVA-to-RTL transpiler.

In checker mode, the generated module exposes `pass` and `fail` outputs for each assertion. The checker-mode flow verifies that the generated hardware checker does not raise false fail signals on the reference FIFO and that the expected fail signals become reachable for the mutated FIFO variants.


## Source files

- `fifo_checker_top.sv`: formal top module. It instantiates the FIFO and the generated checker, defines the reset sequence, and defines assertions and covers on the checker fail/pass signals. Assertions and covers can be enabled with suitable macros.

- `fifo.sby`: SymbiYosys configuration in BMC mode, used to check assertions on checker fail signals.

- `fifo_cover.sby`: SymbiYosys configuration in cover mode, used to check that expected fail/pass signals are reachable.

## BMC check: `fifo.sby`

The default configuration uses bounded model checking with depth 64.

With no mutation enabled, the generated checker is expected not to raise any fail signal:

```tcl
[script]
read -formal -sv -I. -DTEST_ASSERT common.sv fifo.sv fifo_checker.sv fifo_checker_top.sv
```

Run:

```bash
sby -f fifo.sby
```

A mutation can be enabled by adding the corresponding macro. For example:

```tcl
[script]
read -formal -sv -I. -DTEST_ASSERT -DMUT_WR_CNT common.sv fifo.sv fifo_checker.sv fifo_checker_top.sv
```

In this case, the BMC check is expected to fail. A counterexample trace is generated by SymbiYosys.


## Cover check: `fifo_cover.sby`

In cover mode, each mutation is checked by covering the expected checker fail signal. This makes the mutation test pass when the bug is detected.

The mutation-to-assertion coverage is:

```text
MUT_WR_PTR   -> covered by a1
MUT_RD_PTR   -> covered by a3
MUT_WR_CNT   -> covered by a5 and a7
MUT_RD_CNT   -> covered by a8
MUT_EMPTY    -> covered by a5
MUT_FULL     -> covered by a7
MUT_RD_DATA  -> covered by a5 and a6
```

For example, to check that `MUT_WR_PTR` is detected by assertion `a1`, use:

```tcl
[script]
read -formal -sv -I. -DMUT_WR_PTR -DCHECK_A1 common.sv fifo.sv fifo_checker.sv fifo_checker_top.sv
```

Run:

```bash
sby -f fifo_cover.sby
```
