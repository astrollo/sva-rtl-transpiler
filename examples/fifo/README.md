# FIFO example

This example shows the use of the SVA-to-RTL transpiler on a small set of assertions applied to a synchronous FIFO.

## Source files

- `common.sv`: defines `DATA_SIZE` and `FIFO_LOCATIONS`.

- `fifo.sv`: FIFO RTL used as DUT. The file includes optional mutations used to validate checker detectability. Mutations can be enabled at compile time by defining one of the following macros:

  - `MUT_WR_PTR`: disables the write pointer increment.
  - `MUT_RD_PTR`: disables the read pointer increment.
  - `MUT_FULL`: introduces a bug in the `full` flag.
  - `MUT_EMPTY`: introduces a bug in the `empty` flag.
  - `MUT_WR_CNT`: introduces a bug in the occupancy counter on write transactions.
  - `MUT_RD_CNT`: introduces a bug in the occupancy counter on read transactions.
  - `MUT_RD_DATA`: introduces a bug in the read-data path.

- `fifo_assertions.sv`: original concurrent SystemVerilog Assertions to be transpiled.

- `fifo_assertions_immediate.sv`: assertions generated by translating the original concurrent assertions into immediate assertions. This file is intended for formal checking with Yosys/SymbiYosys.

- `fifo_checker.sv`: synthesizable hardware checker generated by the transpiler. This module exposes assertion `pass` and `fail` outputs.

## Transpilation

The Slang frontend is first used to elaborate the SystemVerilog source and generate the AST in JSON format:

```bash
slang --ast-json-source-info --ast-json design.json fifo_assertions.sv
```

The concurrent assertions are then translated into immediate assertions:

```bash
sva_transpiler design.json -o fifo_assertions_immediate.sv
```

A synthesizable hardware checker is also generated:

```bash
sva_transpiler design.json -o fifo_checker.sv --checker --pass
```

## Formal analysis with SymbiYosys

The `formal/` directory contains the formal top module and `.sby` scripts used to validate the translation into immediate assertions.

The `formal-checker/` directory contains the formal top module and `.sby` scripts used to validate the generated hardware checker.
